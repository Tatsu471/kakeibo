# Phase 5.2: UI/UX 実装仕様まとめ

Phase 5.2 で実装されたアニメーション、フォント、画面構成の正式な仕様書です。
これらは「手触り」と「直感的な操作感」を追求するために定義されました。

---

## 🎹 1. 入力画面（キーパッド）
金額入力時のタクティカルなフィードバックを定義しています。

- **ボタン・フィードバック**: `ScaleTransition` を使用。
  - タップダウン時に `1.0` -> `0.95` に縮小。
  - 離すと `1.0` に戻る。
- **入力数値のアニメーション**: `AnimatedSwitcher` と `SlideTransition` を使用。
  - 新しい数字が表示される際、下から上へ「ひょこっ」とスライドしながらフェードインする。
- **フォント比較実験 (A/B Test)**:
  - **1〜4**: `GoogleFonts.outfit()` （モダン・幾何学的）
  - **5〜0 / 記号**: `GoogleFonts.spaceMono()` （デジタル・計器感）

---

## 📈 2. ホーム画面
視覚的な情報提示と遷移の連続性を定義しています。

- **グラフ描画アニメーション**: `LineChart` + `AnimationController` を使用。
  - 画面読み込み時、左から右へ 1.2 秒かけてラインが伸びる演出。
- **FAB 放射状遷移（Radial Expansion）**: `CustomPainter` と `Stack` を使用。
  - `+` ボタンを押した瞬間、ボタンの色（黄色）が円状に拡大し画面を塗りつぶす。
  - 塗りが完了したタイミングで入力画面へフェード遷移する。

---

## ⏳ 3. 履歴画面
過去の記録を整理し、目的のデータに素早くアクセスするための構成です。

- **日付別アコーディオン**: `ExpansionTile` を使用。
  - 記録を日付ごとにグループ化。
  - **当日分のみ初期状態で展開**し、過去分は閉じておく。
- **カテゴリフィルタ**: `ChoiceChip` を使用。
  - 「すべて」「食費」「交通費」を即座にトグル可能。
- **登場アニメーション**: `TweenAnimationBuilder` を使用。
  - 画面を開いた際、各アコーディオンがわずかに遅延しながら下から上へフェードイン（Staggered Entry）する。
